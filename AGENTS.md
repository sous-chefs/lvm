# AGENTS.md — lvm cookbook

AI agent guidance for working on the `lvm` cookbook.
Read this before making any changes.

---

## What this cookbook does

Resource-only cookbook providing six custom resources for managing LVM
(Logical Volume Manager) on Linux. **No gem dependencies** — all LVM
introspection is done via `lvm <cmd> --reportformat json` (LVM ≥ 2.02.158,
present on all supported platforms).

## Resources at a glance

| Resource | What it manages | Key properties |
|---|---|---|
| `lvm_physical_volume` | `pvcreate` on a block device | `name` (device path) |
| `lvm_volume_group` | `vgcreate` / `vgextend` | `name`, `physical_volumes` |
| `lvm_logical_volume` | `lvcreate` / `lvresize` + optional FS + mount | `name`, `group`, `size`, `filesystem`, `mount_point` |
| `lvm_thin_pool` | Thin-provisioning pool | `name`, `group`, `size`, `chunksize`, `zero` |
| `lvm_thin_pool_meta` | Grows `[pool_tmeta]` + spare | `name` (pool name), `group`, `size`, `persist` |
| `lvm_thin_volume` | Thin-provisioned volume from a pool | `name`, `group`, `pool`, `size`, `filesystem`, `mount_point` |

## Key architectural rules

### Never use `lvm` text output — always use JSON
```ruby
# WRONG — fragile text parsing
shell_out('lvs').stdout.split(...)

# RIGHT — stable JSON
LvmHelper.lvs(->(cmd) { shell_out(cmd) })  # returns Hash keyed by "vg/lv"
```

### All LVM state is queried once per resource converge
`LvmActionHelpers` caches `@current_pvs`, `@current_vgs`, `@current_lvs` via
memoisation. These are reset naturally between Chef runs.

### Resource partials require Chef 17+
`use '_lv_common'` and `use '_lv_filesystem'` are Chef 17 features.
`metadata.rb` enforces `chef_version '>= 17'`.

### All resources use `unified_mode true`
No deferred execution. Resources converge top-to-bottom in a single pass.

### Filesystem grow commands are filesystem-specific — never use fsadm
| Filesystem | Grow command | Takes |
|---|---|---|
| `ext4` | `resize2fs <device>` | device path |
| `xfs` | `xfs_growfs <mount_point>` | **mount point** (not device) |
| `btrfs` | `btrfs filesystem resize max <mount_point>` | **mount point** (not device) |

`fsadm` and `lvresize --resizefs` do **not** support btrfs. Never use them here.

## Workflow for adding a new resource

1. **Add the resource file** in `resources/` using `unified_mode true`
2. **Use partials** — include `use '_lv_common'` if the resource has `group`/`size`;
   include `use '_lv_filesystem'` if it creates a filesystem
3. **Add shared helpers** to `LvmActionHelpers` (not duplicated in the resource)
4. **Write a ChefSpec spec** in `spec/unit/resources/<resource>_spec.rb`
5. **Write a fixture recipe** in `test/fixtures/cookbooks/lvm_test/recipes/`
6. **Write InSpec controls** in `test/integration/<suite>/`
7. **Add a kitchen suite** in `.kitchen.yml`
8. **Document it** in `README.md`
9. **Run the full checklist** (see below)

## Running tests

### Linting (Cookstyle)
```bash
# From cookbook root — uses Chef Workstation's embedded cookstyle
cookstyle .

# Auto-correct safe offenses
cookstyle --auto-correct .
```

### Unit tests (ChefSpec via chef exec)
```bash
# Run all unit tests — no Gemfile needed; uses Chef Workstation embedded gems
chef exec rspec spec/ --format documentation

# Run a single resource spec
chef exec rspec spec/unit/resources/logical_volume_spec.rb
```

### Integration tests (Test Kitchen + InSpec)
```bash
# Run a specific suite on a specific platform
kitchen test thin-rockylinux-9

# Run all suites
kitchen test

# Converge only (no verify)
kitchen converge xfs-rockylinux-10

# Verify only (after converge)
kitchen verify xfs-rockylinux-10
```

## Things NOT to do

- ❌ Do not add `chef_gem` or any compile-time gem installs
- ❌ Do not parse `lvs`/`vgs`/`pvs` text output — always use `--reportformat json`
- ❌ Do not use `whyrun_supported?` — deprecated since Chef 14
- ❌ Do not `require 'chef/mixin/shell_out'` or `include Chef::Mixin::ShellOut` inside `action_class` — already included in `Chef::Provider`
- ❌ Do not use `fsadm` or `lvresize --resizefs` for btrfs — they do not support it
- ❌ Do not use `xfs_growfs` with a device path — it requires a mount point
- ❌ Do not add Berkshelf (`Berksfile`) — this cookbook uses Policyfiles
